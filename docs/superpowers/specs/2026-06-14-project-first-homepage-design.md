# 项目优先的个人主页设计

## 目标

在保留现有个人品牌、研究成果和知乎内容的基础上，将主页重心调整为项目经历。页面需要同时服务招聘方、技术负责人、学术同行和潜在合作伙伴，让访客在前两屏内理解董师周的核心能力、承担的职责和已验证的项目结果。

## 设计方向

采用已确认的“完整案例叙事”方向。三个核心工业项目等权展示，不设置唯一重点项目。每个案例使用一致的信息结构：

1. 项目领域与名称
2. 一句话项目定位
3. 两到三个量化结果
4. 技术标签
5. 挑战、职责、方法、结果

项目内容保持真实、简洁，不补充当前资料无法支持的数据或职责。

## 信息架构

首页内容按以下顺序组织：

1. 顶部导航
2. 个人定位 Hero
3. 项目经历
4. 个人经历与最近消息
5. 论文
6. 知乎随笔
7. 专利与竞赛
8. 其他信息

导航中的“项目”放到主要入口位置。Hero 的主文案强调“研究、工程实现和产业落地”的完整能力，并提供跳转到项目区的主要操作；联系方式和 Google Scholar 保留为次要操作。

## 项目案例

### 大语言模型微调与人类偏好对齐

- 结果：SuperCLUE 国内第六、内部测试集提升 5 点以上、70B+ 分布式训练。
- 挑战：提升模型偏好表现，同时控制偏好数据噪声和训练稳定性。
- 职责：训练算法探索、数据构造与筛选、工具调用流程、错误样例分析和标注工具开发。
- 方法：SFT、DPO、ORPO、SimPO，结合 Streamlit、NeMo 和 Slurm 建立训练与反馈流程。

### 大规模场景下的人脸人体时空聚类

- 结果：聚类指标提升 2 点、减少 90% 以上大乱档、在多个实际项目落地。
- 挑战：跨相机身份混淆、大规模聚类乱档和轨迹断裂。
- 职责：市级及封闭场景时空模型设计、实现和标注辅助工具开发。
- 方法：结合相机 GPS、场景拓扑和轨迹约束，并使用 Tkinter 开发 KNN 身份标注工具。

### 超高客流统计

- 结果：普通场景准确率 95% 以上；4000+ 人次每小时的超高客流场景准确率达到 92%；多次项目 PK 超过友商。
- 挑战：高密度客流、徘徊行为和复杂通道对进出人数判断的干扰。
- 职责：独立完成算法与 SDK 设计开发，并承担现场分析、优化和交付闭环。
- 方法：利用行人轨迹状态与区域关系规避徘徊干扰，针对店铺、街道和楼梯等空间制定策略。

## 视觉系统

延续当前冷灰背景、青绿色主色、珊瑚色强调和轻量毛玻璃风格，不引入新的视觉语言。

- Hero 使用更清晰的职业定位和较短的介绍文案。
- 项目区使用纵向排列的大型案例卡片，避免三个等宽小卡片造成信息拥挤。
- 桌面端每张案例卡片左侧展示标题、摘要、指标和标签，右侧以 2×2 网格展示挑战、职责、方法和结果。
- 指标使用高对比数字，但不做夸张的数据仪表盘。
- 案例编号和英文领域标签用于建立节奏，不使用额外插画或装饰性图形。
- 卡片圆角、边框、阴影和悬浮反馈沿用现有 CSS 变量与交互习惯。

## 响应式

- 桌面端项目案例为左右双栏结构。
- 平板端根据可用宽度调整左右栏比例，保证指标与正文不拥挤。
- 移动端改为单栏：项目简介与指标在前，四段案例信息在后。
- 移动端指标允许换行，项目正文不得横向溢出。
- 顶部导航继续支持横向滚动，项目入口保持易发现。

## 交互与动效

站点保持静态、轻交互，不增加路由、弹窗或复杂筛选。

- 导航锚点和 Hero 主按钮可直接跳转到项目区。
- 项目卡片使用现有滚动入场和轻微悬浮反馈。
- `prefers-reduced-motion` 下关闭非必要动画。
- 所有链接和按钮保留清晰的键盘焦点状态。

## 内容调整

- 项目区从知乎随笔之后移动到 Hero 之后。
- 个人经历与最近消息保留，但视觉权重低于项目案例。
- 论文区排在知乎随笔之前，强化“工程与研究兼具”的定位。
- 知乎文章内容、标题和链接保持不变，只调整所在区块的顺序。
- 联系方式、论文、专利、竞赛和其他信息不删除。

## 验证标准

- 静态测试确认三个项目案例均包含挑战、职责、方法和结果。
- 静态测试确认项目区位于个人经历、论文和知乎随笔之前。
- 现有 11 篇知乎文章及链接测试继续通过。
- 桌面和移动视口检查布局、文字换行、导航、项目指标和视觉层级。
- 浏览器控制台无错误，页面资源加载正常。
- 键盘焦点、低动效模式和不支持背景模糊时的降级样式保持可用。
