# 个人主页动效设计

## 目标

在不改变页面内容结构与当前青绿毛玻璃视觉语言的前提下，增加克制但明显的动态层次，减少页面过于静态和单薄的感受。动效需要帮助建立阅读节奏，而不是争夺内容注意力。

## 动效方向

采用“克制但明显”的方案：

- 背景增加两个大范围、低透明度的缓慢光带，形成持续但不抢眼的环境运动。
- Hero 在页面加载时分层入场：标签、姓名、简介、操作按钮、数据卡片与头像依次出现。
- 页面区块沿用滚动揭示，但增加轻微缩放与模糊恢复，使出现过程更有层次。
- 三个项目案例依次进入视口，案例内部的指标卡与四段叙事轻微错峰出现。
- 指标卡悬浮时上移并强化高光，数字做极轻的缩放反馈。
- 导航、按钮和文章链接增加方向明确的位移或下划线反馈。

## 节奏

- Hero 动画总时长控制在约 900 毫秒内，避免阻碍首屏阅读。
- 滚动揭示使用 600 至 750 毫秒的缓动。
- 项目内部错峰间隔控制在 50 至 90 毫秒。
- 背景光带循环大于 16 秒，移动距离小，不产生明显视差。
- 悬浮反馈控制在 180 至 260 毫秒。

## 技术实现

- 使用 CSS 过渡和伪元素，不增加第三方依赖。
- JavaScript 继续使用现有 `IntersectionObserver`，为项目案例设置索引变量，并在进入视口后添加 `visible`。
- 在 HTML 根节点增加 `js` 类后才启用滚动隐藏状态，避免脚本加载失败导致内容不可见。
- 不支持 `IntersectionObserver` 时立即展示所有观察元素。

## 无障碍与性能

- `prefers-reduced-motion: reduce` 下禁用背景漂移、Hero 入场、滚动位移、缩放和错峰延迟。
- 移动端降低背景动画幅度并缩短卡片位移。
- 动效主要使用 `opacity` 和 `transform`，避免频繁触发布局。
- 不增加鼠标跟随、粒子、强视差或持续数字滚动。

## 验证

- 静态测试确认背景关键帧、Hero 入场、项目错峰与低动效覆盖存在。
- 脚本语法检查通过。
- 桌面浏览器确认首屏分层入场、项目案例错峰与悬浮反馈。
- 移动端确认无横向溢出，动画不阻挡内容。
- 浏览器控制台无错误或警告。
